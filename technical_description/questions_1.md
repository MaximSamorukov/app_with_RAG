## ✅ Закрытые слабые стороны

| # | Проблема | Статус | Решение |
|---|----------|--------|---------|
| 1 | S3 bottleneck | ✅ | Разделение очередей heavy/light, memory budget 512MB, один регион S3+worker |
| 2 | Token refresh race | ✅ | Lock / очередь промисов |
| 3 | Векторный поиск | ✅ | Векторный + FTS (RRF) → фильтрация → cross-encoder → top-5 |
| 4 | Failover провайдера | ✅ | Интерфейс `create(primary, fallback?)`, health check `/api/v1/health` |
| 5 | Аудит чанков | ✅ | Поля `deactivated_by` / `deactivated_at`, bulk endpoint `/chunks/bulk` |
| 6 | Лимит версий инструкций | ✅ | Оставлено 10 версий |
| 7 | Лимиты сессий | ✅ | 200 сообщений, 1 активная сессия, sliding window 10 сообщений в LLM |
